## Week 2 — No-Code Automation ✅

Built 6 real automation projects using Zapier and Make.com:

| Project | Tools | Outcome |
|---------|-------|---------|
| Google Form → Gmail Notification | Zapier | First automation ever built |
| Google Form → Google Sheets | Make.com | Debugged a 404 error independently |
| Google Sheets → Gmail Notification | Zapier | Built in 30 min, zero errors |
| AI Content Generation Workflow | Zapier + AI by Zapier | Topic → AI summary → auto-saved to Sheets |
| Lead Collection System | Zapier | Real sellable service for small businesses |
| Automated Task Reminder | Zapier (Schedule trigger) | Time-based automation, fires daily at 7 AM |

**Key lesson:** A 404 error almost always means a wrong ID or connection — always select IDs from the connected account, never type manually.

## Week 3 — Python Foundations ✅

Shifted from no-code tools into real programming.

**Concepts learned:** variables, data types, if/elif/else, loops, functions, lists, dictionaries, string manipulation, file handling (read/write/append), APIs, JSON parsing.

**Projects built:**
- Calculator with 4 operations and user input
- Lists/dictionaries practice — stored Week 2 projects as structured data
- String manipulation toolkit (capitalize, strip, split, join, f-strings)
- File handling + automated report generator (auto-writes formatted reports to .txt)
- Live weather API fetcher — pulled real-time data using `requests`
- AI text summarizer — code fully written, blocked by API billing/region restrictions across Gemini, OpenAI, and Groq
- **Offline keyword chatbot** — pivoted from blocked APIs, built a 3000-pattern matching system that works with zero internet dependency
- Automatic file organizer — sorts files into folders using `os` and `shutil`, handles real bugs like `FileNotFoundError` and missing directories

**Key lesson:** When external tools repeatedly fail, building a self-contained solution is good engineering — not giving up.
